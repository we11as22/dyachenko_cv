# Отчёт: геометрический анализ сегментов (задание 3)

## Цель

Выполнить детальный геометрический анализ сегментов облаков точек ТЛО и сформировать комплексное описание каждого сегмента для последующей классификации и выбора метода реконструкции.

## Методика

Данные: PLY-файлы из `part_2/data` с полями `x`, `y`, `z`, `scalar_Label`.

Pipeline:

1. загрузка PLY и проверка данных;
2. формирование сегментов по label;
3. удаление мелких сегментов;
4. нормализация координат сегмента;
5. расчёт признаков по разделам 6.1–6.5;
6. формирование итогового описания 6.6;
7. сохранение CSV, JSON и визуализаций.

## 6.1 Плотность точек

Для каждого сегмента вычисляются:

- `mean_nn_distance` — среднее расстояние до ближайшего соседа;
- `nn_uniformity` — коэффициент неоднородности (std/mean);
- `sparse_region_ratio`, `dense_region_ratio` — доли разреженных и сгущённых областей по локальной плотности.

Класс плотности:

- `high` — высокая плотность и равномерное распределение;
- `medium` — средняя плотность;
- `low` — низкая плотность или выраженные разреженные области.

## 6.2 Форма распределения

Используется PCA-анализ ковариационной матрицы:

- `linearity`, `planarity`, `sphericity`, `anisotropy`;
- `dominant_axes` — число доминирующих осей (1/2/3).

Интерпретация:

- одна ось — линейная структура;
- две оси — плоская структура;
- равномерное распределение — объёмная форма.

## 6.3 Структура поверхности

Оцениваются:

- `curvature` — PCA-кривизна;
- `normal_variation` — вариация нормалей;
- `sharp_normal_ratio` — доля точек с резким изменением нормали.

Классы: `smooth`, `weakly_curved`, `strongly_curved`, `complex`.

## 6.4 Согласованность нормалей

Для всех точек сегмента вычисляются нормали Open3D и оценивается согласованность с соседними нормалями.

Классы:

- `high` — плоские/гладкие поверхности;
- `medium` — цилиндрические/сферические;
- `low` — сложная геометрия или шум.

## 6.5 Топологическая связность

Строится граф соседства точек в радиусе связности. Оцениваются:

- число компонент связности;
- доля крупнейшей компоненты;
- число изолированных кластеров;
- среднее число соседей.

Классы целостности: `intact`, `mostly_intact`, `fragmented`.

## 6.6 Итог анализа

Для каждого сегмента формируется поле `summary` — комплексное текстовое описание по всем этапам. Оно сохраняется в `metrics/segments_analysis.csv` и выводится на визуализации.

## Запуск

```sh
cd part_2/task_3
source ~/envs/.venv/bin/activate
python scripts/run_geometric_analysis.py
```
