# Отчёт: адаптивная реконструкция поверхностей по облакам точек ТЛО

## Цель работы

Цель проекта — построить устойчивый pipeline реконструкции поверхностей инженерных объектов по облакам точек лазерного отражения. Данные представлены PLY-файлами с координатами `x`, `y`, `z` и меткой сегмента `scalar_Label`.

## Архитектура алгоритма

Реализованный комплекс выполняет следующие этапы:

1. загрузка всех PLY-файлов из `part_2/data`;
2. проверка формата, наличия координат и `scalar_Label`;
3. удаление некорректных точек и статистических выбросов;
4. опциональное прореживание через voxel downsampling;
5. формирование сегментов по label;
6. удаление сегментов с малым числом точек;
7. расчёт геометрических признаков;
8. классификация сегмента;
9. выбор алгоритма реконструкции;
10. построение mesh-модели сегмента;
11. объединение mesh-сегментов в модель исходного файла;
12. расчёт количественных и качественных метрик.

## Геометрический анализ

Для каждого сегмента вычисляются:

- число точек;
- объём bounding box;
- плотность точек;
- среднее расстояние до ближайшего соседа;
- PCA-признаки формы: `linearity`, `planarity`, `sphericity`, `curvature`;
- согласованность нормалей;
- число компонент связности;
- доля крупнейшей компоненты.

PCA-признаки позволяют отделять вытянутые, плоские и объёмные формы. Согласованность нормалей показывает регулярность поверхности, а связность помогает выявлять разорванные или шумные сегменты.

## Классификация сегментов

Сегмент относится к одному из типов:

- `plane` — плоская поверхность;
- `tube` — трубчатый или вытянутый объект;
- `sphere` — сферическая или объёмно-замкнутая форма;
- `complex` — сложная геометрия.

Классификация основана на сочетании PCA-признаков, кривизны и согласованности нормалей.

## Выбор метода реконструкции

Выбор метода выполняется автоматически:

- для плоских сегментов используется Alpha Shapes, так как метод устойчиво строит поверхность по плоским и слабокриволинейным областям;
- для трубчатых объектов используется Ball Pivoting, так как он хорошо восстанавливает поверхности при равномерной плотности точек;
- для сферических форм используется Poisson Surface Reconstruction, так как он даёт гладкую замкнутую поверхность;
- для сложной геометрии метод выбирается по связности и согласованности нормалей.

## Оценка качества

Количественные метрики сохраняются в `metrics/segments_metrics.csv`:

- `mean_cloud_to_mesh` — среднее отклонение точек сегмента от реконструированной поверхности;
- `p95_cloud_to_mesh` — 95-й процентиль отклонения;
- `max_cloud_to_mesh` — максимальное отклонение;
- `accuracy_ratio` — доля точек, находящихся близко к поверхности;
- `mesh_vertices`, `mesh_triangles` — размер итоговой модели;
- `mesh_components` — число компонент mesh.

Качественно анализируются артефакты, гладкость поверхности, связность и соответствие исходному облаку через PNG-визуализации в `visualizations/`.

## Запуск

```sh
cd part_2/task_2
source ~/envs/.venv/bin/activate
pip install -r requirements.txt
python scripts/run_reconstruction.py
```

Для быстрой проверки:

```sh
python scripts/run_reconstruction.py --max-files 1 --max-points-per-segment 1500
```

По умолчанию обработка выполняется последовательно. При необходимости она распараллеливается по исходным PLY-файлам параметром `--workers`, например `python scripts/run_reconstruction.py --workers 4`. Каждый файл реконструируется независимо, поэтому параллельный запуск не меняет логику алгоритма и ускоряет полный прогон по датасету.
Индикатор tqdm обновляется после завершения файла, поэтому для контроля длительных операций скрипт дополнительно печатает строки `[start]` и `[done]` по каждому обрабатываемому PLY-файлу.
Для устойчивости каждый PLY по умолчанию обрабатывается в отдельном дочернем процессе. Это важно для Open3D Poisson-реконструкции: ошибки уровня `Segmentation fault` возникают внутри C++-кода библиотеки и не перехватываются обычным Python `try/except`. При изоляции такой сбой помечает только один файл как `failed`, а общий прогон продолжает обработку. Метрики сохраняются после каждого файла.

## Сравнение подходов

Для проверки преимущества адаптивного выбора предусмотрен режим принудительного метода. Он позволяет запустить один и тот же pipeline с фиксированным алгоритмом реконструкции и сравнить результаты с адаптивной схемой:

```sh
python scripts/run_reconstruction.py --force-method poisson --output-dir outputs_poisson --metrics-dir metrics_poisson --visualization-dir visualizations_poisson
python scripts/run_reconstruction.py --force-method alpha_shape --output-dir outputs_alpha --metrics-dir metrics_alpha --visualization-dir visualizations_alpha
python scripts/run_reconstruction.py --force-method ball_pivoting --output-dir outputs_bpa --metrics-dir metrics_bpa --visualization-dir visualizations_bpa
```

Сравнение выполняется по метрикам `mean_cloud_to_mesh`, `p95_cloud_to_mesh`, `accuracy_ratio`, числу успешных реконструкций и числу компонент mesh. Адаптивный подход считается более качественным, если при сопоставимом числе треугольников он даёт меньшее среднее отклонение, большую долю близких к поверхности точек и меньше артефактов на визуализациях.

Сравнение выполнено на одинаковой выборке из первых 50 PLY-файлов (450 сегментов). Все подходы завершились без ошибок.

| Подход | Сегментов | Ошибок | Mean cloud-to-mesh | Mean p95 | Accuracy ratio | Компонент mesh | Треугольников |
|--------|-----------|--------|--------------------|----------|----------------|----------------|---------------|
| Adaptive | 450 | 0 | 1.227 | 2.896 | 0.918 | 4.66 | 5836 |
| Poisson only | 450 | 0 | **1.160** | **2.721** | **0.922** | 5.31 | 7192 |
| Alpha Shapes only | 450 | 0 | 2.046 | 6.493 | 0.884 | 59.04 | 935 |
| Ball Pivoting only | 450 | 0 | 1.207 | 2.825 | 0.911 | **1.88** | 1358 |

По численным отклонениям на этой выборке лучшим оказался фиксированный Poisson, но он строит более тяжёлые модели. Ball Pivoting даёт близкую ошибку при существенно меньшем числе треугольников и лучшей связности mesh. Alpha Shapes оказался самым слабым универсальным методом: ошибка выше, а число компонент сильно растёт. Адаптивный подход занимает промежуточное положение: он автоматически выбирает метод под тип сегмента и сохраняет устойчивость на полном наборе из 500 файлов, где было реконструировано 4440 сегментов без ошибок.

## Дополнительные направления развития

В рамках расширения проекта возможны следующие улучшения:

- машинное обучение для классификации сегментов вместо эвристических правил;
- ускорение алгоритмов через downsampling, батчевую обработку и параллельный запуск по файлам;
- обработка в реальном времени для потоков ТЛО;
- BIM-интеграция: экспорт реконструированных поверхностей и метаданных сегментов в формат, пригодный для инженерных моделей;
- автоматическая постобработка mesh: закрытие дыр, сглаживание, удаление мелких компонент.

## Выводы

Разработанная программа обрабатывает облака точек, формирует сегменты по label, анализирует их геометрию и автоматически выбирает метод реконструкции. По сравнению с применением одного базового метода ко всем сегментам адаптивная схема должна повышать устойчивость: плоские, трубчатые, сферические и сложные участки обрабатываются разными алгоритмами, лучше соответствующими их форме.

## Соответствие техническому заданию

- Загрузка данных: читаются все PLY-файлы из `part_2/data`, проверяются формат, поля и размеры.
- Предобработка: выполняется удаление выбросов, нормализация сегментов и опциональный downsampling.
- Формирование сегментов: точки группируются по `scalar_Label`, мелкие сегменты отбрасываются.
- Геометрический анализ: считаются плотность, PCA-форма, структура поверхности, нормали и связность.
- Классификация: сегменты относятся к плоским, трубчатым, сферическим или сложным.
- Выбор метода: автоматически выбирается Poisson, Alpha Shapes или Ball Pivoting.
- Реконструкция: для каждого сегмента строится mesh, ошибки изолируются на уровне сегмента/файла.
- Сборка модели: mesh-сегменты объединяются в итоговую модель исходного облака.
- Оценка качества: сохраняются отклонения от облака, точность, связность и размеры mesh.
- Результаты: сохраняются модели, визуализации, метрики и отчёт.
